Next.js provides Metadata APIs that can be used to add metadata tags to your pages. These APIs are:

1. [The static `metadata` object](#static-metadata)
2. [The dynamic `generateMetadata` function](#dynamic-metadata)
3. Special [file conventions](/docs/app/api-reference/file-conventions/metadata) that can be used to add static or dynamically generated OG images and favicons.

## Static metadata

To define static metadata, export a [`Metadata` object](/docs/app/api-reference/functions/generate-metadata#metadata-object) from a static [`layout.js`](/docs/app/api-reference/file-conventions/layout) or [`page.js`](/docs/app/api-reference/file-conventions/page) file. For example, to add a title and description to the blog route:

```tsx filename="app/blog/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'My Blog',
  description: '...',
}

export default function Page() {}
```

```jsx filename="app/blog/layout.tsx" switcher
export const metadata = {
  title: 'My Blog',
  description: '...',
}

export default function Page() {}
```

Behind the scenes, Next.js will automatically generate the relevant `<head>` elements for your pages. You can view a full list of [available options here](/docs/app/api-reference/functions/generate-metadata#metadata-fields).

## Dynamic metadata

You can use [`generateMetadata`](/docs/app/api-reference/functions/generate-metadata) function to `fetch` metadata that depends on external data. For example, to fetch the title for a blog post:

```tsx filename="app/blog/[slug]/page.tsx" switcher
import type { Metadata, ResolvingMetadata } from 'next'

type Props = {
  params: Promise<{ id: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}

export async function generateMetadata(
  { params, searchParams }: Props,
  parent: ResolvingMetadata
): Promise<Metadata> {
  const slug = (await params).slug

  // fetch post information
  const post = await fetch(`https://api.vercel.app/blog/${slug}`).then((res) =>
    res.json()
  )

  return {
    title: post.title,
    description: post.description,
    openGraph: {
      url: `https://example.com/blog/${slug}`,
      images: [{ url: post.imageUrl, width: 1800, height: 1600 }],
    },
  }
}

export default function Page({ params, searchParams }: Props) {}
```

```jsx filename="app/blog/[slug]/page.js" switcher
export async function generateMetadata({ params, searchParams }, parent) {
  const slug = (await params).slug

  // fetch post information
  const post = await fetch(`https://api.vercel.app/blog/${slug}`).then((res) =>
    res.json()
  )

  return {
    title: post.title,
    description: post.description,
    openGraph: {
      url: `https://example.com/blog/${slug}`,
      images: [{ url: post.imageUrl, width: 1800, height: 1600 }],
    },
  }
}

export default function Page({ params, searchParams }) {}
```

To learn more about the available options, see the [API Reference](/docs/app/api-reference/functions/generate-metadata).

## Favicons

Favicons are small icons that represent your site in bookmarks and search results.

To add a favicon to your application, create a `favicon.ico` and add to the root of the app folder.
